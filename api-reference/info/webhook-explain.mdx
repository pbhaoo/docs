该接口为 Basswallet 平台主动向业务方发起回调通知请求。

### 回调请求
  - 平台回调的请求与商户请求接口中的签名方式和数据结构保持一致。
  - 回调类型（提现订单或充值订单）可通过回调内容中的 `webhookType` 字段判断，不同类型回调的内容不同。
  - 一笔订单会有两次回调。根据回调内容中的 `status` 字段判断：
    - 当 `status` 为 `WaitingConfirmation` 时，为第一次回调。
    - 当 `status` 为 `Success` 或 `Fail` 时，为第二次回调。

### 回调响应
  - 对于回调，您需要使用不带正文的 HTTP 状态码 200 进行响应。如果服务器暂时不可用或者响应状态不是 200，我们将随着延迟时间的增加多次重新发送回调；重新发送次数是有限的。超过一定次数后回调将失败，如果需要重发，您需要在商户后台的“交易回调”中进行手动推送。

### 回调测试
  您可以使用 [requestcatcher.com](https://requestcatcher.com) 工具方便地进行回调测试。

### 为什么需要两次回调？
  - 区块链转账在首次被打包进区块时，回调可以快速告知客户交易情况，但此时并不代表交易一定成功。由于区块链网络可能因分叉等因素回滚交易，首次回调的交易可能无效。因此，只有当交易经过一定的区块确认数后，才会进行第二次回调确认交易成功。

业务方需明确知悉回调的规则，根据回调内容做出正确的业务处理。如需了解具体规则，请点击 [这里](https://example.com) 了解更多。
